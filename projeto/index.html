Esse trabalho apresentará o gerenciador de versões Git e suas funções.

Git é o sistema de controle de versão open source mais usado no mundo atualmente! Ele é usado para controlar o histórico de alterações de arquivos e principalmente de projetos de desenvolvimento de software. Ele permite mais flexibilidade no fluxo de trabalho, segurança e desempenho.

Junto com o GitHub (repositório), o git pode oferecer uma estrutura simples para que códigos de um projeto de software possa ser salvo e acessado em segurança.

Agora vamos aos comandos mais utilizados:

Comando Função
init Cria um novo repositório do Git, pode ser utilizado na criação de um projeto.
clone Tem como função copiar os arquivos de um repositório remoto para o repositório local
branch É um comando em que se cria uma ramificação a partir do código principal, sem alterá-lo
checkout Checkout pode variar de funcionalidade, entre variar de branches ou recuperar dados de um ponto já salvo.
status Tem como função mostrar algumas informações sobre o repositório operante. Por exemplo, os arquivos que foram atualizados.
diff Comando que mostra a diferença que foram feitas no commit, diretórios e entre outros.
add Tem como função adicionar alterações nos arquivos, que futuramente serão incluídos no próximo commit.
commit Commit tem como funcionalidade atualizar as alterações feitas no código, gerando um nova versão.
push Comando utilizado para enviar as alterações feitas em um repositório remoto.
pull É utilizado para sincronizar o repositório local com as alterações salvas no repositório remoto.
revert Reverte alterações anteriores, mas sem excluir o que foi alterado, apenas voltando no ponto anterior.
merge Usado para sincronizar mudanças de um branch selecionado com o branch sendo utilizado
stash Salva de forma temporária ações não concluídas no diretório, podendo reverter esse estado temporário.
rm Utilizado para excluir arquivos do Git. Podendo marcar o que será excluído antes de atualizar o código.
config Comando para configurar o Git. Por exemplo, definir a configuração global, local e entre outras, personalizando o Git.
reset Comando para desfazer ou mudar o histórico de commits, podendo mudar a ordem em que as atualizações foram feitas.
remote Comando que comanda as conexões com o repositório remoto, podendo adicionar ou excluir alterações remotas ao Git.
fetch Utilizado para verificar as alterações mais recentes, permitindo a vizualização das modificações feitas por outras pessoas.
show Exibe informações específicas sobre um commit específico. Detalhes como alterações realizadas, autor e entre outras.
help Comando que informa sobre como utilizar um outro comando Git.
rebase Utilizado para reorganizar ou modificar o histórico dos commits, podendo assim, deixar de forma mais linear e compreensível as alterações feitas.
blame Rastreia quem fez alterações em cada linha do código, permitindo saber os autores das alterações feitas.
tag Comando utilizado para marcar pontos específicos no Git, permitindo uma clareza na identificação de atualizações importantes.

Ao entrar no site do GitHub, fazer sua conta e, posteriormente, o login, o próprio github recomenda a criação de um repositório.

É possível também controlar o acesso ao repositório com as seguintes opções:

  1. - Privado:
    2. Acesso permitido somente aos usuários que são listados como permitidos.
  2. - Público
    3. Qualquer usuário do GitHub
